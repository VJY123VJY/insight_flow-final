================================================================================
    PROCESSING FLOW (NEW)
================================================================================

BEFORE: JWT → Replay → Rate Limit → ALLOW

AFTER:  JWT → [MDU VALIDATION] → Replay → Rate Limit → ALLOW
                  (NEW - CRITICAL)

MDU Validation Steps:
  1. Extract dataset_id from request
  2. Query MDU: GET /api/v1/datasets/canonical/{dataset_id}
  3. Validate trust_level is TRUSTED or VERIFIED
  4. Return dataset metadata or reject with error
  5. Log all validation decisions

Response now includes dataset metadata:
  - dataset_id
  - dataset_trust_level (TRUSTED/VERIFIED/PROVISIONAL)
  - dataset_schema
  - dataset_compatibility
  - dataset_provenance

================================================================================
    ERROR HANDLING (All Scenarios Covered)
================================================================================

200 OK                  - Dataset allowed (TRUSTED/VERIFIED)
400 Bad Request         - Missing dataset_id
401 Unauthorized        - Invalid JWT token
403 Forbidden           - Insufficient trust level (PROVISIONAL)
404 Not Found           - Dataset not in MDU
429 Too Many Requests   - Rate limit exceeded
502 Bad Gateway         - Invalid MDU response
503 Service Unavailable - MDU down (fail-closed)
